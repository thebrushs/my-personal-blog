### 1.2 发现的问题与建议修改

#### 🔴 **严重问题（必须修改）**

**问题1：用户表缺少Web后台登录密码字段**
- **位置**：3.1.1 用户表（user）
- **问题描述**：文档中Web管理后台需要"账号密码登录"，但用户表设计只有`openid`字段，没有密码相关字段
- **建议修改**：
  ```
  增加字段：
  - password_hash: VARCHAR(255) - 密码哈希值（Web后台登录用）
  - salt: VARCHAR(64) - 密码盐值
  - login_type: TINYINT - 登录类型（1-小程序微信登录，2-Web账号密码登录，3-两者都绑定）
  ```

**问题2：数据同步冲突处理机制不明确**
- **位置**：4.1.1 数据同步模块
- **问题描述**：文档提到"离线同步"和"多端同时操作"，但没有详细说明冲突处理策略
- **建议修改**：在功能模块中补充：
  ```
  数据同步冲突处理策略：
  1. 乐观锁机制：记录表增加version字段，每次更新时检查版本号
  2. 冲突检测：最后修改时间戳比对，后提交者提示"数据已更新，请刷新"
  3. 离线同步策略：本地记录操作日志，联网后按时间顺序依次同步
  ```

#### 🟡 **中等问题（建议修改）**

**问题3：AI标题生成功能细节不足**
- **位置**：4.1.1 AI标题生成模块
- **问题描述**：只提到"调用服务端AI接口"，没有说明具体的输入输出格式
- **建议修改**：补充接口设计：
  ```
  AI标题生成接口设计：
  - 输入：图片URL或文字内容（至少一项）
  - 输出：生成的标题（限制128字符内）
  - 失败处理：生成失败时返回默认标题"学习记录+日期"
  - 超时设置：接口超时时间5秒，超时后使用默认标题
  ```

**问题4：图片上传限制不够具体**
- **位置**：七、MVP版本边界
- **问题描述**：提到"单张5MB以内，批量上传最多5张"，但没有说明支持的图片格式
- **建议修改**：
  ```
  图片上传限制：
  - 支持格式：JPG、PNG、GIF（不支持WebP，小程序兼容性问题）
  - 单张大小：≤5MB
  - 批量上传：最多5张
  - 图片压缩：上传前小程序端自动压缩至宽度≤1080px
  ```

**问题5：数据库表缺少版本控制字段**
- **位置**：3.1 核心表结构
- **问题描述**：为了实现乐观锁和冲突检测，需要在核心表增加版本字段
- **建议修改**：
  ```
  study_record表增加字段：
  - version: INT - 数据版本号（乐观锁用，每次更新+1）
  ```

#### 🟢 **轻微问题（可选修改）**

**问题6：Web后台数据导出格式未明确**
- **位置**：4.2.1 内容管理模块
- **问题描述**：提到"导出记录为Excel/JSON"，但没有说明具体导出内容
- **建议修改**：
  ```
  数据导出规范：
  - Excel格式：包含日期、标题、内容类型、内容详情、创建时间等列
  - JSON格式：完整数据结构，包含关联的内容项
  - 导出范围：支持按日期范围、内容类型筛选后导出
  ```

**问题7：小程序离线缓存容量限制未说明**
- **位置**：4.1.2 小程序适配要点
- **问题描述**：提到"核心记录本地缓存"，但没有说明缓存容量和清理策略

  - 版本升级策略：新版本接口上线后，旧版本至少保留3个月
  - 版本兼容：v1和v2可同时运行，通过路由区分

## 二、技术选型建议

### 2.1 服务端框架选择建议
文档中提到"Spring Boot（Java）/ Node.js（Express）"，建议明确选择：

**推荐：Node.js + Express + TypeScript**
- 理由：
  1. 前后端技术栈统一（Vue3 + TypeScript）
  2. 开发效率高，适合快速迭代
  3. 异步IO适合处理图片上传等IO密集型任务
  4. 部署简单，资源占用少

### 2.2 数据库选择建议
文档提到"MySQL + Redis"，这个选择合理，建议补充：
- MySQL版本：5.7+ 或 8.0+（支持JSON字段）
- Redis用途：用户Session、热点数据缓存、接口限流
